1.1 Besoin initial
------------------

Le besoin de depart etait de concevoir une application web permettant
d'acceder simplement a des informations boursieres, avec une presentation
adaptee a un public non expert.

Constat initial :
- Les plateformes financieres classiques sont souvent complexes pour un
  utilisateur debutant.
- L'utilisateur a besoin d'un indicateur de synthese pour orienter sa lecture
  des donnees (score simplifie).
- Il faut centraliser recherche, actualites et informations de compte dans une
  interface unique et lisible.

Le projet devait donc proposer :
- une recherche d'action (nom ou ticker),
- un resultat interpretable rapidement (score, rendement, risque),
- une experience utilisateur claire sur ordinateur et mobile.


1.2 Objectifs fonctionnels
--------------------------

Objectifs principaux atteints par la solution :

1) Recherche d'actions
   - Recherche par nom d'entreprise ou symbole boursier.
   - Recuperation des donnees marche via API externe.

2) Analyse simplifiee
   - Calcul d'un score simplifie de 0 a 10.
   - Affichage des indicateurs rendement/risque.
   - Affichage d'actualites financieres associees.

3) Gestion des comptes
   - Inscription, connexion, deconnexion.
   - Session securisee avec JWT.
   - Page "Mon compte" avec statut utilisateur.

4) Gestion des favoris
   - Ajout/suppression de valeurs en watchlist.
   - Consultation de la liste des favoris.

5) Version premium
   - Parcours d'abonnement via Stripe Checkout.
   - Mise a jour du statut premium en base de donnees (webhook Stripe).

6) Donnees et qualite
   - Persistance PostgreSQL (avec mode memoire de secours).
   - Tests automatises unitaires et d'integration API.


1.3 Type de projet
------------------

Nature du projet :
- Type : creation complete d'une solution applicative web.
- Cadre : projet de formation BTS SIO SLAM (E6).
- Evolution : enrichissements progressifs (favoris, S&P 500, Stripe, tests,
  documentation).

Caracteristiques techniques :
- Architecture client-serveur (frontend statique + API backend Express).
- Integration de services externes (Finnhub, RSS Yahoo Finance, Stripe).
- Base relationnelle PostgreSQL.

Resultats attendus (et obtenus) :
- Application exploitable localement, stable et demonstrable.
- Fonctions principales operationnelles (recherche, auth, favoris, premium).
- Documentation technique exploitable pour l'oral E6.

2.2 Environnement de developpement
----------------------------------

Environnement type utilise :

- Systeme d'exploitation : Windows 10/11
- IDE / editeur : Visual Studio Code / Cursor
- Runtime : Node.js (>= 18)
- Gestionnaire de paquets : npm
- Base de donnees : PostgreSQL
- Outils de tests : Jest + supertest
- Navigateur de test : Chrome / Edge / Firefox
- Gestion de versions : Git + GitHub

Configuration locale :
- Variables d'environnement dans .env
- Separation stricte des secrets (non versionnes)
- Lancement local via npm start


2.3 Technologies utilisees
--------------------------

Frontend :
- HTML5
- CSS3
- JavaScript (vanilla)

Backend :
- Node.js
- Express
- Middlewares : CORS, JSON parser
- Authentification : JWT
- Hashage mot de passe : Argon2

Donnees :
- PostgreSQL via driver pg
- SQL relationnel (tables, contraintes, requetes parametrees)

Services externes :
- Finnhub (donnees boursieres)
- Yahoo Finance RSS (actualites)
- Stripe (abonnements premium)

Qualite :
- Tests unitaires et integration API (Jest + supertest)
- Documentation technique versionnee
